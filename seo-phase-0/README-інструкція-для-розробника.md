# Вулик — SEO Фаза 0: інструкція для розробника

Пакет технічних правок для сайту https://www.vulyk.clinic/ (статичний HTML).
Мета — прибрати базові SEO-проблеми, щоб сайт краще індексувався та показувався в Google.
Порядок виконання — від найважливішого. Орієнтовний обсяг: 1 робочий день.

Файли в цій папці:
- `sitemap.xml` — готовий, залити в корінь сайту.
- `jsonld-schema.html` — блоки структурованих даних для вставки в `<head>`.
- цей README — решта правок (meta, title, OG, soft-404, H1).

---

## 🔴 ПРІОРИТЕТ 1 — критичні правки

### 1.1. Виправити soft-404 (найважливіше)
**Проблема:** будь-який неіснуючий URL (напр. `/zzz.html`, `/privacy.html`) віддає
**головну сторінку зі статусом 200 OK**. Google сприймає це як сотні дублів головної →
розмивається індекс, марнується краулінговий бюджет.

**Рішення** (залежно від хостингу — схоже на Cloudflare Pages / статик-хост):
- Створити справжню сторінку `404.html` (коротка: лого, «Сторінку не знайдено»,
  кнопки на Головну та Запис).
- Налаштувати хостинг віддавати її з **HTTP-статусом 404**, а не 200.
  - Cloudflare Pages: файл `404.html` у корені підхоплюється автоматично зі статусом 404.
  - Якщо стоїть SPA-редірект «усе → index.html» — прибрати його для неіснуючих шляхів.
- Перевірка після фіксу: відкрити `https://www.vulyk.clinic/будь-що-неіснуюче` →
  у DevTools → Network статус має бути **404**.

### 1.2. Залити sitemap.xml
- Покласти `sitemap.xml` (з цієї папки) у корінь → доступний як
  `https://www.vulyk.clinic/sitemap.xml`.
- У Google Search Console: **Індексування → Файли Sitemap** → додати URL.
- Дописати в `robots.txt` рядок:
  ```
  Sitemap: https://www.vulyk.clinic/sitemap.xml
  ```

### 1.3. Meta description на КОЖНУ сторінку
Зараз тег `<meta name="description">` **відсутній** → Google сам ліпить сніпет.
Додати в `<head>` кожної сторінки унікальний опис 140–160 символів із ключами + містом + CTA.

Приклади (готові до вставки):

```html
<!-- index.html -->
<meta name="description" content="Сімейна амбулаторія «Вулик» у Львові: педіатрія, сімейний лікар, УЗД, вузькі спеціалісти. Доказова медицина, партнер НСЗУ. Запис у Telegram/Viber.">

<!-- sub-pediatriya.html -->
<meta name="description" content="Дитячий лікар у Львові в амбулаторії «Вулик»: огляди, щеплення, супровід розвитку дитини. Досвідчені педіатри, запис онлайн. Стрийська 86В, Тичини 21.">

<!-- sub-simeyna-med.html -->
<meta name="description" content="Сімейний лікар у Львові: декларація з НСЗУ безкоштовно, профогляди, ведення хронічних станів. Амбулаторія «Вулик» — доказова медицина. Записатися.">

<!-- dir-uzi.html -->
<meta name="description" content="УЗД у Львові для дітей, вагітних і дорослих в амбулаторії «Вулик»: серце, щитоподібна, органи, суглоби. Сучасне обладнання. Запис у месенджері.">

<!-- sub-casd.html -->
<meta name="description" content="Діагностика розладів аутистичного спектру (CASD) у дітей 1–17 років у Львові. Амбулаторія «Вулик»: точний скринінг, дитячий психіатр. Записатися.">

<!-- prices.html -->
<meta name="description" content="Ціни на послуги амбулаторії «Вулик» у Львові: консультації лікарів, УЗД, аналізи, діагностика. Актуальний прайс. Багато послуг — безкоштовно за НСЗУ.">
```

**Шаблон для решти сторінок послуг:**
`[Послуга] у Львові в амбулаторії «Вулик»: [1–2 переваги/деталі]. [Спеціаліст/обладнання]. Запис онлайн.`

## 🟠 ПРІОРИТЕТ 2 — важливі правки

### 2.1. Один `<h1>` на сторінку
На головній зараз **6 тегів `<h1>`** (заголовок + 5 карток новин). Правило: **рівно один H1**.
- Головна: залишити H1 = «Медицина, де людина — найвища цінність».
- Заголовки карток новин/блоку перевести на `<h2>` або `<h3>`.
- Перевірити всі сторінки: один H1 = головна тема сторінки (з ключем, напр. «Педіатрія у Львові»).

### 2.2. Розширити `<title>` містом
Заголовки є, але без гео. Додати «Львів» до ключових комерційних сторінок:
- `Педіатрія — Вулик` → `Дитячий лікар (педіатр) у Львові — Вулик`
- `Ультразвукова діагностика — Вулик` → `УЗД у Львові — Вулик`
- `Стрийська, 86В — Вулик` → `Амбулаторія на Стрийській, 86В, Львів — Вулик`
- Головна: `Вулик — Медична клініка` → `Вулик — сімейна амбулаторія у Львові | педіатрія, УЗД, сімейний лікар`
Довжина title — до ~60 символів, щоб не обрізалось у видачі.

### 2.3. Open Graph + canonical у `<head>` кожної сторінки
Зараз немає OG → некрасиві прев'ю в Telegram/Viber/Facebook. Шаблон:

```html
<link rel="canonical" href="https://www.vulyk.clinic/sub-pediatriya.html">
<meta property="og:type" content="website">
<meta property="og:site_name" content="Вулик — сімейна амбулаторія">
<meta property="og:locale" content="uk_UA">
<meta property="og:title" content="Дитячий лікар (педіатр) у Львові — Вулик">
<meta property="og:description" content="…той самий текст, що в meta description…">
<meta property="og:url" content="https://www.vulyk.clinic/sub-pediatriya.html">
<meta property="og:image" content="https://www.vulyk.clinic/assets/og-pediatriya.jpg">
<meta name="twitter:card" content="summary_large_image">
```
`og:image` — 1200×630 px. Можна зробити 1 універсальний банер бренду, якщо немає часу на окремі.

### 2.4. Вставити JSON-LD (файл `jsonld-schema.html`)
- Блок 1 → у `<head>` `index.html`.
- Блок 2 → у `<head>` `location-striyska.html`.
- Блок 3 → у `<head>` `location-tychyny.html`.
- Заповнити поля `ЗАПОВНИТИ`: координати (з Google Maps), к-ть і оцінку Google-відгуків.
- Перевірити: https://search.google.com/test/rich-results

## 🟡 ПРІОРИТЕТ 3 — дрібні, але корисні

### 3.1. Alt до зображень
На головній 5 зображень без `alt`. Додати описовий alt усім значущим `<img>`
(напр. `alt="Педіатр оглядає дитину в амбулаторії Вулик"`). Декоративні — `alt=""`.

### 3.2. Створити реальні сторінки-заглушки, на які вже є посилання
- `privacy.html` (Політика конфіденційності) — **обов'язково юридично**, бо форма запису
  й месенджери збирають персональні дані. Google/довіра теж це люблять.
- Розглянути `terms.html` / умови обробки даних.

### 3.3. Підготувати сторінку локації Лінкольна
У меню/новинах згадується Лінкольна, 4 (відкриття восени 2026), але сторінки немає.
Коли буде контент — створити `location-lincolna.html` і додати в sitemap + schema.
